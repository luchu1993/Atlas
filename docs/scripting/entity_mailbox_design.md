# 设计: Entity Mailbox 代理机制

> 归属阶段: ScriptPhase 4 (~~Atlas.Generators.Rpc~~ → Atlas.Generators.Def) | 关联: ScriptPhase 3 (ClrScriptEngine)

> **⚠ 部分内容已过时 (2026-04-14)**:  
> 本文档中引用的 `[ClientRpc]`/`[CellRpc]`/`[BaseRpc]` Attribute 和 `Atlas.Generators.Rpc` 项目已在 [DefGenerator 统一重构](../DEFGEN_CONSOLIDATION_DESIGN.md) 中删除。  
> Mailbox 代理现由 **DefGenerator** 从 `.def` 文件的 `<client_methods>`/`<cell_methods>`/`<base_methods>` 节生成。核心设计思想不变，仅输入来源从 C# Attribute 改为 `.def` 文件。

---

## 1. 背景

BigWorld 架构中，实体分布在 Base / Cell / Client 三端，通过 **Mailbox** 进行跨端透明 RPC 调用：

```python
# BigWorld Python 风格
entity.client.showDamage(100, position)       # → 拥有者客户端
entity.allClients.playAnimation("attack")     # → 所有客户端
entity.otherClients.showEffect("slash")       # → 其他客户端
entity.cell.moveTo(Vector3(10, 0, 20))        # → Cell 实体（可能在另一个进程）
entity.base.saveData()                        # → Base 实体（可能在另一个进程）
```

Python 的实现依赖 `__getattr__` 动态代理——**完全依赖运行时反射**，是 IL2CPP 禁区。

## 2. C# 方案：Source Generator 编译期 Mailbox 代理

核心思路：为每种 RPC 方向生成一个 **`readonly struct` 代理类型**，包含与实体相同的方法签名，方法体自动序列化参数并发送。

### 2.1 用户代码

```csharp
[Entity("Avatar")]
public partial class Avatar : ServerEntity
{
    // Server → Client
    [ClientRpc]
    public partial void ShowDamage(float amount, Vector3 hitPos);

    [ClientRpc]
    public partial void PlayAnimation(string animName);

    [ClientRpc]
    public partial void ShowEffect(string effectName);

    // Base → Cell (跨进程)
    [CellRpc]
    public partial void MoveTo(Vector3 position);

    [CellRpc]
    public partial void CastSkill(int skillId, uint targetId);

    // Cell → Base (跨进程)
    [BaseRpc]
    public partial void SaveData();

    [BaseRpc]
    public partial void OnCellDestroyed();

    // Client → Cell (exposed in .def)
    [CellRpc]  // .def: exposed: OwnClient
    public partial void RequestMove(Vector3 target);

    [CellRpc]  // .def: exposed: OwnClient
    public partial void RequestUseItem(int itemId);
}
```

### 2.2 Source Generator 输出

#### 2.2.1 Client Mailbox 代理

```csharp
// <auto-generated by Atlas.Generators.Rpc/>

/// Client Mailbox — 强类型代理，仅包含 [ClientRpc] 方法。
/// readonly struct 零堆分配，方法可内联。
public readonly struct AvatarClientMailbox
{
    private readonly Avatar _entity;
    private readonly MailboxTarget _target;

    internal AvatarClientMailbox(Avatar entity, MailboxTarget target)
    {
        _entity = entity;
        _target = target;
    }

    public void ShowDamage(float amount, Vector3 hitPos)
    {
        var writer = new SpanWriter(24);
        try
        {
            writer.WriteFloat(amount);
            writer.WriteVector3(hitPos);
            _entity.SendClientRpc(
                RpcIds.Avatar_ShowDamage, _target, writer.WrittenSpan);
        }
        finally { writer.Dispose(); }
    }

    public void PlayAnimation(string animName)
    {
        var writer = new SpanWriter(64);
        try
        {
            writer.WriteString(animName);
            _entity.SendClientRpc(
                RpcIds.Avatar_PlayAnimation, _target, writer.WrittenSpan);
        }
        finally { writer.Dispose(); }
    }

    public void ShowEffect(string effectName)
    {
        var writer = new SpanWriter(64);
        try
        {
            writer.WriteString(effectName);
            _entity.SendClientRpc(
                RpcIds.Avatar_ShowEffect, _target, writer.WrittenSpan);
        }
        finally { writer.Dispose(); }
    }
}
```

#### 2.2.2 Cell / Base Mailbox 代理

```csharp
// <auto-generated/>

public readonly struct AvatarCellMailbox
{
    private readonly Avatar _entity;
    internal AvatarCellMailbox(Avatar entity) => _entity = entity;

    public void MoveTo(Vector3 position)
    {
        var writer = new SpanWriter(12);
        try
        {
            writer.WriteVector3(position);
            _entity.SendCellRpc(RpcIds.Avatar_MoveTo, writer.WrittenSpan);
        }
        finally { writer.Dispose(); }
    }

    public void CastSkill(int skillId, uint targetId)
    {
        var writer = new SpanWriter(8);
        try
        {
            writer.WriteInt32(skillId);
            writer.WriteUInt32(targetId);
            _entity.SendCellRpc(RpcIds.Avatar_CastSkill, writer.WrittenSpan);
        }
        finally { writer.Dispose(); }
    }
}

public readonly struct AvatarBaseMailbox
{
    private readonly Avatar _entity;
    internal AvatarBaseMailbox(Avatar entity) => _entity = entity;

    public void SaveData()
    {
        _entity.SendBaseRpc(
            RpcIds.Avatar_SaveData, ReadOnlySpan<byte>.Empty);
    }

    public void OnCellDestroyed()
    {
        _entity.SendBaseRpc(
            RpcIds.Avatar_OnCellDestroyed, ReadOnlySpan<byte>.Empty);
    }
}
```

#### 2.2.4 接收端分发

```csharp
// <auto-generated/>

internal static partial class RpcDispatcher
{
    // CellApp 收到 CellRpc 时的分发（含 exposed 方法）
    private static void Dispatch_Avatar_CellRpc(
        Avatar target, int rpcId, ref SpanReader reader)
    {
        switch (rpcId)
        {
            case RpcIds.Avatar_RequestMove:
            {
                var targetPos = reader.ReadVector3();
                target.OnRequestMove(targetPos);
                break;
            }
            case RpcIds.Avatar_RequestUseItem:
            {
                var itemId = reader.ReadInt32();
                target.OnRequestUseItem(itemId);
                break;
            }
        }
    }

    // 客户端收到服务端 ClientRpc 时的分发（Unity 侧使用）
    private static void Dispatch_Avatar_ClientRpc(
        Avatar target, int rpcId, ref SpanReader reader)
    {
        switch (rpcId)
        {
            case RpcIds.Avatar_ShowDamage:
            {
                var amount = reader.ReadFloat();
                var hitPos = reader.ReadVector3();
                target.OnShowDamage(amount, hitPos);
                break;
            }
            case RpcIds.Avatar_PlayAnimation:
            {
                var animName = reader.ReadString();
                target.OnPlayAnimation(animName);
                break;
            }
            case RpcIds.Avatar_ShowEffect:
            {
                var effectName = reader.ReadString();
                target.OnShowEffect(effectName);
                break;
            }
        }
    }
}
```

### 2.3 使用体验

```csharp
// 服务端脚本 — 与 BigWorld Python 完全等价
[Entity("Avatar")]
public partial class Avatar : ServerEntity
{
    [Replicated]
    private Vector3 _position;

    protected override void OnTick(float dt)
    {
        // 与 Python entity.client.showDamage(100, pos) 等价
        this.Client.ShowDamage(100f, this.Position);  // Position 由 Generator 生成

        // 与 Python entity.allClients.playAnimation("attack") 等价
        this.AllClients.PlayAnimation("attack");

        // 与 Python entity.otherClients.showEffect("slash") 等价
        this.OtherClients.ShowEffect("slash");

        // 与 Python entity.cell.moveTo(pos) 等价
        this.Cell.MoveTo(new Vector3(10, 0, 20));

        // 与 Python entity.base.saveData() 等价
        this.Base.SaveData();
    }
}
```

## 3. 与 Python `__getattr__` 的对比

| 维度 | Python `__getattr__` | C# Source Generator |
|------|---------------------|-------------------|
| 方法名拼写错误 | 运行时报错 | **编译期报错** |
| 参数类型错误 | 运行时报错 | **编译期报错** |
| 参数数量错误 | 运行时报错 | **编译期报错** |
| IDE 自动补全 | 不支持 | **完美支持** |
| 跳转到定义 | 跳到 `__getattr__` | **跳到实体方法定义** |
| 性能 | 字典查找 + `*args` boxing | **零开销, struct 方法内联** |
| 堆分配 | 每次创建代理对象 | **零堆分配 (readonly struct)** |
| IL2CPP 兼容 | 不适用 | **完全兼容** |
| 序列化 | 运行时动态 | **编译期精确生成** |

## 4. 跨进程 Mailbox（分布式透明调用）

Cell/Base Mailbox 的跨进程调用对脚本完全透明：

```
BaseApp 进程                             CellApp 进程
┌────────────────┐                      ┌────────────────┐
│ Avatar (Base)  │                      │ Avatar (Cell)  │
│                │                      │                │
│ this.Cell      │                      │ OnMoveTo()     │
│   .MoveTo(pos) │                      │ {              │
│      │         │                      │   // 业务逻辑  │
│      ▼         │                      │ }              │
│ SendCellRpc()  │                      │      ▲         │
│      │         │                      │      │         │
└──────┼─────────┘                      └──────┼─────────┘
       │                                       │
       ▼                                       │
  C++ Bundle → Channel → TCP → Channel → C++ dispatch_rpc()
                                               │
                                          C# RpcDispatcher
                                            .Dispatch()
```

Source Generator 生成的 `SendCellRpc()` 内部调用 `NativeApi`，C++ 引擎层根据实体的 Cell 地址将消息路由到正确的 CellApp 进程。

## 6. 基类中的发送方法

```csharp
// ServerEntity 基类提供的 RPC 发送基础设施
public abstract class ServerEntity
{
    // 子类的 Mailbox 代理会调用这些方法
    protected internal void SendClientRpc(
        int rpcId, MailboxTarget target, ReadOnlySpan<byte> payload)
    {
        NativeApi.SendClientRpc(EntityId, rpcId, (byte)target, payload);
    }

    protected internal void SendCellRpc(int rpcId, ReadOnlySpan<byte> payload)
    {
        NativeApi.SendCellRpc(EntityId, rpcId, payload);
    }

    protected internal void SendBaseRpc(int rpcId, ReadOnlySpan<byte> payload)
    {
        NativeApi.SendBaseRpc(EntityId, rpcId, payload);
    }
}
```
