# AI LOD（AI Level of Detail）

> **用途**：定义 Atlas 中怪物 / NPC AI 的性能分层策略——按距离与战斗状态动态调整 AI 复杂度，让 5000 并发服务器同时承载几千个 AI 实体且玩家感觉"世界是活的"。
>
> **读者**：AI 设计（必读）、工程（必读）、战斗策划（§3、§5 必读，理解远距离怪物表现限制）。
>
> **状态**：草案 v0.1 — 待团队评审。
>
> **前置文档**：`OVERVIEW.md`、`AI_ARCHITECTURE.md`、`DENSITY_ADAPTIVE_NETWORKING.md`、`CELL_ARCHITECTURE.md`
>
> **关联文档**：`BOSS_AI.md`（boss 不走标准 LOD）、`MOVEMENT_SYNC.md`（远端 AI 移动）

## 1. 设计目标与边界

### 1.1 问题

5000 并发服务器集群可能存在：
- 数千 NPC 巡逻 / 闲逛（任务给予者、商人、训练师）
- 几百怪物在不同地图练级点
- 几十 boss / 精英怪物
- 玩家附近的几十战斗中怪物

如果**所有 AI 都跑完整行为树**：
- 单怪物 BT tick ~30 μs × 数千怪物 = 几十毫秒/秒
- 寻路 / 视线检测开销爆炸
- 跨 cell 通信压力大

无法承载。**必须分层**。

### 1.2 玩家应该感受到的

✅ **靠近怪物**：行为完整、智能、反应灵敏
✅ **中距离**：怪物在做事（巡逻 / 交谈 / 战斗），不是雕像
✅ **远距离**：能看到"那边有怪在动"，感受世界活着
✅ **远到看不见**：完全不存在也无所谓

### 1.3 玩家不应该感受到的

❌ **走近时怪物突然"从睡眠中醒来"**（看到状态切换瞬间）
❌ **远处怪物完全静止**（世界感觉死气沉沉）
❌ **怪物群聚集时性能崩溃**（卡顿）
❌ **AI"作弊"看到不该看到的玩家**（穿墙感知）

### 1.4 设计目标

1. **性能可承载**：数千 AI 实体不卡服
2. **玩家无感**：LOD 切换玩家察觉不到
3. **世界活感**：远处也有活动迹象
4. **资源高效**：CPU 用在玩家附近的 AI 上
5. **行为一致**：远 / 近怪物**最终行为**一致（玩家走近不会"换性格"）

## 2. LOD 分层

### 2.1 五档 LOD

按怪物到**最近玩家**的距离分层：

| LOD 档 | 距离 | AI 频率 | 行为深度 | 玩家感受 |
|---|---|---|---|---|
| **L0 战斗** | 玩家在战斗范围 | 10 Hz BT | 完整 | 智能、灵敏 |
| **L1 警觉** | < 30 m 但未战斗 | 5 Hz BT | 完整 | 注意到玩家 |
| **L2 活跃** | 30–80 m | 2 Hz BT | 简化（无群组协调） | 在做日常 |
| **L3 远景** | 80–200 m | 0.5 Hz | 仅"伪行为"（巡逻 + 站立切换） | 远处有动静 |
| **L4 休眠** | > 200 m | 几乎不跑 | 完全冻结 | 不可见 |

### 2.2 战斗状态覆盖距离

**战斗中怪物无视距离一律 L0**：
- 玩家走开了 100m，但战斗未结束 → 怪物仍跑 L0 BT
- 怪物追击玩家进入超出 200m 范围 → 仍 L0
- 战斗结束（target 死亡 / 失去目标 5 秒以上）→ 自然降级

### 2.3 群组关联

群组中任一成员战斗 → 整个群组升级到 L0（参见 `AI_ARCHITECTURE.md §6`）。
群组不能"一半在打架一半在睡觉"。

### 2.4 重要 NPC 不降级

剧情 NPC / 商人 / 关键 NPC 永远不降到 L4：
- 玩家可能突然找他们
- 不能"找不到 NPC"

设计为 `lod_floor: L2`——最低也是 L2。

### 2.5 Boss 特殊处理

Boss 永远 L0（参见 `BOSS_AI.md §3`）。Boss 战 AI 复杂度高，不能简化。

## 3. 各档行为详解

### 3.1 L0 战斗

完整行为：
- 行为树每 100 ms tick
- 群组协调
- 完整感知（视觉 / 听觉 / 触觉）
- 寻路（必要时）
- 全部技能可用

玩家感受：
- 智能、有挑战
- 反应灵敏
- 战斗有意义

### 3.2 L1 警觉

未战斗但注意到玩家：
- BT 每 200 ms tick（5 Hz）
- 完整感知（保持警觉）
- 简化决策（先看看再行动）
- 不主动攻击（等玩家进入战斗范围或自己被攻击）

玩家感受：
- 怪物**看到了我**（视觉 telegraph：怪物转向玩家方向、呼吸加快）
- 走近会触发战斗（紧张感）
- 走开会缓和（怪物逐渐放松）

### 3.3 L2 活跃

中距离怪物：
- BT 每 500 ms tick（2 Hz）
- **不做群组协调**（独立行动）
- 感知简化（只检查视觉，不听不触）
- 可执行：巡逻、闲逛、与其他 NPC 交互
- **不释放技能**（远距离释放无意义，浪费）
- 寻路只在必要时（如撞墙后）

玩家感受：
- 远处看到怪物在巡逻
- 怪物在做日常（卫兵站岗、农夫种田）
- 世界**有活力**

### 3.4 L3 远景

更远的怪物：
- BT 每 2 秒 tick（0.5 Hz）
- **伪行为**：从预设的"行为快照"中随机选择
  - 站立 / 行走 / 转身 / 互相看
- 不感知玩家（除非很近触发感知）
- 不寻路（仅按预设巡逻路线移动）
- 不与玩家交互

玩家感受：
- 远处看到"小人在动"
- 走得方向 + 简单姿势变化
- 不会"突然冲过来"（远到那个程度玩家不期望复杂行为）

### 3.5 L4 休眠

玩家完全看不见的怪物：
- BT 完全停止
- 位置 / HP / 状态保留（玩家走近时恢复）
- 不参与任何仿真
- 仅在 timer 触发时苏醒（如复活）

玩家感受：
- 不可见
- 走近时**自动从 L4 恢复到 L3 / L2**

### 3.6 LOD 切换

玩家走近怪物：
- 距离 > 200m → L4
- 200m → 80m：L4 → L3（"苏醒"）
- 80m → 30m：L3 → L2（开始活跃）
- 30m → 战斗范围：L2 → L1 → L0

切换是连续的，**玩家感觉不到瞬间状态变化**：
- L4 → L3：怪物开始"伪动作"（与之前一致的姿势 + 简单移动）
- L3 → L2：恢复完整 idle 动画 + 开始查看周围
- L2 → L1：注意到玩家，转向警觉

### 3.7 切换的视觉处理

苏醒过程要**自然**：
- 不要"睁开眼睛"动画（会暴露 LOD 系统）
- 直接接续之前的姿势（站立或巡逻）
- 玩家**不应感受到"被怪物"**

如果走近发现"远处那个站着不动的怪物突然开始巡逻"——LOD 暴露。

## 4. 性能影响

### 4.1 各档 CPU 开销

| LOD | 单怪物每秒 CPU | 1000 怪物总开销 |
|---|---|---|
| L0 战斗 | ~300 μs/s | 300 ms/s |
| L1 警觉 | ~150 μs/s | 150 ms/s |
| L2 活跃 | ~50 μs/s | 50 ms/s |
| L3 远景 | ~10 μs/s | 10 ms/s |
| L4 休眠 | < 1 μs/s | < 1 ms/s |

### 4.2 现实分布估算

5000 并发玩家场景中典型 AI 分布：
- L0：~200 怪物（玩家正在战斗）= 60 ms/s
- L1：~400 怪物（玩家附近）= 60 ms/s
- L2：~800 怪物（玩家远视野）= 40 ms/s
- L3：~1500 怪物（地图远景）= 15 ms/s
- L4：~5000 怪物（沉睡）= 5 ms/s

**总 ~180 ms/s = 18% 单核 CPU**——一个 CellApp 跑得动。

### 4.3 寻路预算

寻路开销大（1–10 ms/次），LOD 限制：
- L0：可寻路（任意需要时）
- L1 / L2：寻路结果缓存使用，必要时重新寻路（< 1/min）
- L3 / L4：禁止寻路（仅按预设路径移动）

### 4.4 数据库 / 持久化

L4 怪物完全不参与服务器仿真，但**状态保留在内存**：
- HP（被打死的状态）
- buff / debuff
- 玩家造成的伤害记录（影响掉落）

苏醒时数据完整。

## 5. 玩家走近时的体验

### 5.1 渐进唤醒

```
玩家距离 250 m: 怪物 L4 (沉睡，不可见)
玩家距离 200 m: 怪物 L4 → L3 (开始 "存在")
                视觉：远端实体进入 AoI（参见 DENSITY_ADAPTIVE_NETWORKING.md L4 层）
                  
玩家距离 150 m: 怪物 L3 (远景动作)
                视觉：看到小人偶尔走动
                
玩家距离 80 m:  怪物 L3 → L2 (恢复完整动作)
                视觉：怪物在做日常，可看清动作
                
玩家距离 30 m:  怪物 L2 → L1 (注意到玩家)
                视觉：怪物转向玩家，警觉姿势
                
玩家距离 < 战斗范围: 怪物 L1 → L0 (战斗模式)
                视觉：怪物攻击玩家
```

### 5.2 隐藏 LOD 边界

切换瞬间避免"行为跳变"：
- L3 → L2：动画从"伪 idle" 切到"完整 idle"——动画 transition 200 ms 平滑
- L2 → L1：动画切到 alert 姿势——同样平滑
- L1 → L0：开始攻击——视觉上是"怪物决定攻击"，自然

如果玩家观察到"我走过 L 边界时怪物切到不同动作"——LOD 暴露。

### 5.3 远端实体订阅与 LOD 联动

**客户端 LOD**（参见 `DENSITY_ADAPTIVE_NETWORKING.md §7`）和 **AI LOD****协调**：
- 客户端订阅档 L0 / L1 / L2 / L3 / L4 ←→ AI LOD 档对应

不完全一致，但**通常协调**：
- 客户端不订阅（实体不在视野）→ AI 可以是 L4 休眠
- 客户端 L4（远景）→ AI L3（远景行为足够）
- 客户端 L0（贴身）→ AI L0（战斗）

### 5.4 跨 cell 边界的 AI

怪物可能在另一 cell（玩家看到的 Ghost 数据）：
- 玩家所在 cell 不直接控制怪物 AI
- 怪物 AI 在它的 real cell 跑
- 但 LOD 由"距离玩家最近距离"决定（跨 cell 距离）

参见 `GHOST_ENTITY.md §4`——Ghost 同步频率与 AI LOD 也需要协调。

## 6. 边缘场景

### 6.1 多个玩家同时影响 LOD

100 米外有玩家 A，5 米内有玩家 B：
- 怪物的 LOD = 取最高（按最近距离）
- A 看到的远景 + B 看到的战斗——同一只怪
- B 的视角下完整 BT
- A 看到的也是同一行为（通过 ghost / snapshot 同步）

### 6.2 跨 LOD 的群组

群组成员散布不同距离：
- 群组中一员 L0 战斗 → **整组 L0**（如 §2.3）
- 防止"群组成员 BT 不一致"

### 6.3 玩家瞬移

玩家用闪现 / 传送：
- 离开当前区域 → 旧区域怪物逐渐降 LOD
- 进入新区域 → 新区域怪物逐渐升 LOD
- **不需立即切**——会让玩家瞬移瞬间感受到"怪物突然出现"

延迟切换（5–10 秒过渡）让 LOD 变化平滑。

### 6.4 怪物追击玩家越界

怪物追击玩家进入"远离群组的位置"：
- 仍 L0（战斗中）
- 不与原群组共享感知
- 战斗结束后回归原群组（巡逻回去）

### 6.5 长时间无玩家访问的区域

没玩家走动的偏远地图：
- 所有怪物 L4
- 不消耗 CPU
- "活的" 状态保留（玩家来访时恢复）

### 6.6 地图传送 / 副本进出

玩家进入副本：
- 主图玩家位置改变 → 影响主图怪物 LOD
- 副本怪物从 0 启动（独立 instance）
- 玩家退副本 → 副本逐渐销毁 → 主图怪物按位置自适应

## 7. 与世界感（World Living）的平衡

### 7.1 "世界活着" 的设计意图

MMO 玩家进入游戏，**期望感受到世界活力**：
- 远处城堡上空鸟群飞过
- 远处森林里似乎有动物
- 远处战斗的声光（远征军？）
- 主城里 NPC 走动 / 摆摊

如果都"沉睡"，世界感觉死气沉沉。

### 7.2 L3 / L4 的"伪行为"

让远处 / 沉睡怪物**看起来像活的**：
- L3 怪物按预设路径偶尔走动（即使没在跑 BT）
- L4 怪物有"环境动画"（鸟群飞翔、风吹草动）—— 这些是装饰，不是 AI
- 主城 NPC 即使没玩家也"日常活动"（站岗 / 端茶）

### 7.3 装饰性生物（Ambient creatures）

某些生物**完全不参与战斗**，只是装饰：
- 远处的飞鸟
- 路边的小动物
- 城里的猫 / 狗

这些用**纯动画系统**（不需要 AI）：
- 预设路径 + 动画播放
- 客户端本地生成（服务端不发送 / 仿真）
- 可能仅 fake snapshot（客户端按位置生成）

省 CPU + 添氛围。

### 7.4 远景战斗的"声光"

远处战斗（如远征军、攻城、boss 战）：
- 服务端广播"远景事件"（声光通知）
- 玩家屏幕看到"远处烟雾"或"远处闪光"
- 玩家可走过去看具体战斗
- 走近时自然 LOD 升级

让 MMO 感觉到"世界发生事件"，不只是"我个人的故事"。

## 8. 测试与调优

### 8.1 LOD 性能测试

P4 阶段必须做：
- 服务器 5000 怪物 + 100 玩家压测
- 监控 CellApp CPU、内存、tick 耗时
- 验证 LOD 切换玩家不感知

### 8.2 玩家感受测试

P3+ 盲测：
- 玩家走近 / 走远怪物，观察是否感觉"突变"
- 玩家在远处看世界是否"活的"
- 玩家追逐怪物时是否平滑

### 8.3 调优杠杆

- 各档距离阈值（30 / 80 / 200 m）
- 各档 BT 频率
- 切换过渡时长

不停服 hot reload 配置。

### 8.4 监控

- 各 LOD 档怪物数量分布
- LOD 切换频率
- 玩家走近时 LOD 升级耗时
- AI CPU 占比

## 9. FAQ 与玩家关注问题

### Q1: AI LOD 会让远处怪物"作弊"看到我吗？

不会——感知有距离限制，远处怪物（L3 / L2）**不会**因为 LOD 简化而获得超能力。

实际上 L3 怪物**完全不感知玩家**，更不会作弊。

### Q2: 我远离战斗中的怪物，它会"消失" 吗？

不会——战斗中保持 L0。怪物会追你或回归原位，但不会消失。

直到怪物失去目标 5 秒，才降级 LOD。

### Q3: 远处怪物看起来"动作迟缓"是 LOD 吗？

可能是。L2 / L3 怪物 BT tick 频率低，反应不敏捷。
但**距离远（视觉小）+ 动作慢** 玩家通常不察觉违和。

如果玩家发现违和，调整切换距离或动作频率。

### Q4: 城里 NPC 永远不睡觉吗？

主要 NPC 永远 L2+（参见 §2.4）。
但**装饰 NPC** 可能临时降级（玩家不在主城时）。

切换平滑，玩家进城时一秒内恢复完整。

### Q5: 怪物等级会影响 LOD 吗？

不会——LOD 仅基于距离。Boss 例外（永远 L0）。

低级怪和精英怪在同距离下 LOD 一致，但精英怪**性格** / **技能** 不同（行为差异通过 BT 实现，不通过 LOD）。

### Q6: 服务器闲时（半夜）AI 性能如何？

闲时玩家少，怪物大多 L4 → L3，CPU 占用极低。
反而有"能量"做其他事（数据备份、地图清理等）。

### Q7: PvP arena 内有 LOD 吗？

PvP arena 通常无 NPC 怪物。如果有（召唤物 / 助战兽）：
- 仅 8 玩家小场景
- 召唤物全部 L0（保证清晰反馈）
- 不需要 LOD

### Q8: 玩家 AFK 在主城会让 NPC LOD 降级吗？

不会——主城 NPC 设计上 lod_floor: L2，不降到 L3 / L4。

否则玩家 AFK 后再回来，NPC "突然活过来" 体验差。

### Q9: 同一群组成员距离玩家不一样，LOD 怎么决定？

群组**取最高 LOD**：群组任一成员靠近玩家 → 整组按那个成员的距离决定 LOD。

例：5 个哥布林散布，离玩家最近的 20m → 整组 L1（警觉）。

### Q10: 玩家可以"利用 LOD" 吗（比如远程射杀沉睡怪物）？

不可以——AI LOD 是性能优化，不是设计漏洞。
玩家**永远看不见 L4 怪物**（客户端不订阅），自然无法攻击。

L3 / L2 怪物正常订阅，被攻击立即触发战斗（升 L0）。

---

### 反模式清单

- ❌ LOD 切换无过渡（瞬间动作变化）
- ❌ 让远处怪物完全静止（世界死气沉沉）
- ❌ 战斗中也降级 LOD（玩家被打着却"AI 不在思考"）
- ❌ 群组成员各自独立 LOD（行为不一致）
- ❌ 重要 NPC 也降到 L4（玩家找不到）
- ❌ LOD 仅基于距离不考虑战斗状态（错杀战斗中怪物）
- ❌ 客户端订阅与 AI LOD 不协调（看到的怪物 AI 实际没跑）
- ❌ "苏醒"动画暴露 LOD 系统
- ❌ Boss 也走 LOD（机制可能丢失）
- ❌ 远处装饰物用完整 AI（白白浪费 CPU）
